# 步骤 9：自审检查清单

> 加载时机：PRD 生成完成后，逐项检查质量时使用。

生成 PRD 后，逐项检查：

## 结构检查
- [ ] PRD 一级模块是否只包含：版本管理、需求背景、流程图、功能清单、详细需求、验收用例？
- [ ] 版本管理是否符合规则：评审时 V1.0，评审后修改 V1.1，开发/测试 bug 单变更继续递增？
- [ ] 需求背景是否同时说清业务背景和业务目标？
- [ ] 流程图章节是否包含图片引用和文字描述？简单字段类需求是否也输出了简化流程图图片？

## 需求范围检查
- [ ] 如果瓦力需求中有删除线标记的项，确认这些项没有被写入 PRD 正文
- [ ] 需求描述中的每个有效改动点都在 PRD 中有对应描述吗？（逐项核对，不遗漏）

## 行为层检查
- [ ] 每一行描述，用户/PM 能看懂吗？不需要技术背景？
- [ ] 有没有混入代码、内部接口定义、数据库、技术术语？（外部对接类例外：接口字段表、参数枚举值、校验规则允许出现）

## 完整性检查
- [ ] 每个操作都有成功/失败的反馈描述吗？
- [ ] 每个有条件展示的元素都写了展示条件吗？
- [ ] 每个操作按钮都写了权限要求吗？
- [ ] 跨模块联动是双向描述的吗？（A→B 和 B 收到后做什么）
- [ ] 历史数据需要处理吗？（新字段对已有数据的影响）

## 页面覆盖检查
- [ ] 所有受影响的页面都列出来了吗？有没有漏掉的页面？（列表改了，详情页要不要联动？导出要不要包含新字段？）
- [ ] 每个页面的变更描述，前端开发能直接对应到要改的组件吗？
- [ ] 页面间的联动写清楚了吗？（A 页面操作后，B 页面怎么刷新/更新）

## 外部对接检查（仅外部对接类 PRD）
- [ ] 外部对接内容是否收敛在"流程图"和"详细需求"中，没有新增"数据流向/异常处理/兼容现有功能"一级章节？
- [ ] 每个功能模块是否包含：功能描述 + 操作流程 + 接口契约？
- [ ] 接口字段表是否完整：参数名 + 说明 + 枚举值 + 必填/可选 + 校验规则？
- [ ] 枚举值是否列全了？（中英文映射、取值范围、状态码含义）
- [ ] 校验规则是否用中文描述？（不写代码，写"XX字段必填，范围0~90天"）
- [ ] 错误消息是否定义了具体的 msg 内容？（不是"返回错误"，而是"返回 msg：XXX"）
- [ ] 字段映射表是否完整？（我方字段 ↔ 对方字段，含转换说明）
- [ ] 异常场景是否在对应详细需求功能点中覆盖：接口超时、数据不一致、必填缺失？
- [ ] 兼容现有功能是否在对应详细需求功能点中列全？（工单、开票、导出、违章等）
- [ ] 是否仍然没有代码片段、HTTP 方法、内部 API 设计？

## 一致性检查
- [ ] 功能清单和详细需求一致吗？清单中每一行在详细需求中都有对应展开吗？
- [ ] 枚举值全文一致吗？（同一个状态在不同地方叫法一样？）
- [ ] 模块间联动关系完整吗？（触发方和被触发方都描述了？）

## 原型检查
- [ ] 所有有 UI 变更的页面都生成了原型吗？（或标注了"原型待补充"及原因）
- [ ] 无 UI 变更的功能点，原型列标注了"无 UI 变更"吗？
- [ ] 原型中新增/修改的字段与 PRD 说明列一致吗？
- [ ] 原型使用的模拟数据是否使用了真实枚举值（来自 Hub）？
- [ ] 原型是否展示了完整页面？（能看出是哪个页面/弹框）
- [ ] 未变更区域的字段是否与 Vue 源码一致（标签、顺序、结构）？
- [ ] 原型中是否有源码中不存在的额外组件/装饰元素？（有则删除）
- [ ] **组件形态是否保持一致**？（新增信息是在原有组件上扩展，不是替换成另一种组件）
- [ ] **页面上是否有标注装饰**？（橙色边框、"PRD新增"标签、灰色删除线 → 全部删除，变更说明放在底部文档区）
- [ ] **删除的字段是否完全移除**？（不同时展示旧字段和新字段）
- [ ] **新增字段的 props 是否与同级组件一致**？（collapse-tags、clearable、filterable 等配置对齐）

---

发现问题立即修正，不输出未通过自审的 PRD。
